# How to Use This Pet Medication Index

## Why the pages look similar

The pages in this project are intentionally structured like reference notes, not articles. A pet owner in the middle of a medication question often needs fast orientation: What should I have ready? Who should I call? What information belongs on the bottle label? What details should not be guessed?

Each medication page uses repeated headings so an owner can compare questions without treating the pages like medical instructions. Similar headings do not mean similar decisions. Two pets with the same medication name may still need completely different veterinary review.

## Start with the situation

If the owner is holding a bottle and wondering whether more medication is available, start with the refill and label page. If the owner noticed vomiting, diarrhea, swelling, breathing issues, collapse, unusual tiredness, or sudden behavior change, start with the side-effect watch page and contact the veterinary office or emergency service as appropriate.

If the owner is planning travel, use the travel checklist before the final week. If the medication is flavored, compounded, refrigerated, or in a liquid form, use the compounded medication page and the medication-specific note.

## Use the question lists

The question lists are meant to prevent vague calls. “My pet needs more medicine” is less useful than: “This is a refill for Bella, a dog. The bottle label says the refill number is zero. The veterinary office last saw her two weeks ago. Should the pharmacy request authorization, or does the veterinarian need a recheck first?”

When someone calls the pharmacy counter, the practical details matter. Have the bottle nearby before calling. Read the pet name, medication name, form, refill number, and any storage note from the label.

## Use the data files

The data files are included so the project behaves like a GitHub reference repository. The medication data file lists slugs, categories, species notes, veterinarian questions, pharmacy questions, and page paths. The glossary file keeps terms short and consistent.

The data files are not hidden medical instructions. They are index aids for navigation, maintenance, and checklist creation.

## Know the boundary

The pharmacy can help identify the next step, but the veterinary office owns the care decision. If the question is about diagnosis, suitability, timing changes, stopping, restarting, adverse signs, or whether the medication still fits the pet, call the veterinary office.

## A pharmacy note on consistency

Because the pages are built as reference notes, a repeated heading is a safety feature. It keeps the owner from missing the difference between a veterinarian question and a pharmacy question. The same bottle label can lead to different next steps depending on whether the concern is refill status, storage, a changed instruction, or a pet that seems unwell.

Do not use the data files as shortcuts around the veterinary office. They are a navigation layer for the repository, not a replacement for the pet’s chart.
